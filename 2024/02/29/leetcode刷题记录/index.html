·给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。你可以假设数组中无重复元素。

​ 思路：该题的难点在于，当数组中不存在target，应该把target插在哪个位置上呢？（用哪个变量指示失配位置呢？）
​ 经过实践，在使用while(left<=right)时，当最终跳出循环时（失配时）right=left-1，而mid是等于left还是right取决于最后失配时nums[mid]是大于target还是小于target：当nums[mid]>target时，要更改right，而在此之前left=right=mid（失配的前一步是肯定要走到这里的，也就是while(left<=right)里=条件的最后一次）所以right=mid-1，那么此时left=mid；当nums[mid]<target时，要更改left，所以left=mid+1，right=mid；使用几组案例就可以发现，left的位置刚好就是target应该插入的位置。

2.移除元素（双指针）

·leetcode27：
·要求：删除数组中等于这个目标值的元素并且返回新数组的大小
·vector提供了erase()函数来实现删除某个位置元素的操作，其操作复杂度是O(n)的，要将后续的元素整体向前移/覆盖。与此同时，再次对该向量使用size()返回的长度会减一，但实际上，数组的实际’容量‘并没有改变，只是’长度‘被- -，最后面因为删除元素所造成的’额外‘位置只是被忽略了，没有做实际的处理。（理解 ‘容量capacity’ 和 ‘长度length/size’ 之间的差别）

①暴力解法：O(n^2)

②双指针: O(n)
快指针：新数组所需要的值
慢指针：新数组的下标
快指针不断寻找符合的元素，慢指针不断为新来的元素指明位置。有点像在编一个麻花绳，快指针在前面乱绳处不断缕齐，慢指针在后面将快指针缕齐的地方编好。

·把slow++写在nums[slow++]=nums[fast]里面会快3ms！直接击败100%用户。

4.长度最小数组（双指针、滑动窗口）

·leetcode209：
·给定一个含有 n 个正整数的数组和一个正整数 target 。找出该数组中满足其总和大于等于 target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

·暴力解法：O(n^2)

·在break这里想了很久，为什么要直接break出去呢：因为对于当前的外循环，起点i是固定的，j每次++，那么所要找的最短的一定是最先符合的子序列，所以后续j++只能使得字串变得更长了，所以应该直接break掉（有点和滑动窗口弄混了…）

·滑动窗口
在暴力解法中，第一个for循环（外层循环）滑动窗口的起始位置，第二个循环（内层循环）滑动窗口的终止位置。
而滑动窗口只使用一个for循环来完成。如果滑动窗口使用一个for循环来表示起始位置，那么只可能是一个一个不断地遍历之后的位置来获得最短子序列，那么就陷入了暴力解法的怪圈，因此一定要使用一个for循环表示滑动窗口的终止位置。
那么如何移动窗口起始位置呢？（滑动窗口的精髓）当当窗口的值大于s，那么窗口就应该缩小了（起始位置向后移动）。滑动窗口的精妙之处在于根据当前子序列的大小的情况，不断调节子序列的起始位置。

·对于滑动窗口的算法复杂度：O(n)，为啥不是o(n^2)呢?明明有两个嵌套的循环啊。
要看每个元素的操作次数，每个元素进入滑动窗口操作一次，出滑动窗口操作一次，因此每个元素操作了两次，是2 X n，因此就是O(n)。而对暴力解法而言，对于每一个起始位置，都要整个遍历终止位置，是n X n的。

·在while(sum>=s)这里想了很久，为什么有个=呢？如果等于的话则说明当sum等于s的话也要进行一次’起始位置向后移动‘的操作，那不就是说此时所得到的结果会比预期小了吗？！
例如：s=7，nums =[2,3,1,2,4,3]。最短子序列是[4,3]，最后一次4，3满足sum=s，则start++不是指向3返回1了？？
是因为我没有把while内的工作内容掌握全面，流程掌握清楚。注意看代码的实现流程，是先记录的sublength的长度，后移动的start，也就是说，当我们发现了某处满足了sum>s的条件，我们要向后移动起始位置的start了，但是移动后很有可能不满足了sum>s的条件，因此一定要首先保存subLlength然后再移动start，这样做是合理正确的，那么当sum=s时，尽管start在最后确实向后移动了一位，但是subLength的计算在start移动之前，因此并不影响最终结果的正确性。只要将前面的顺序搞明白，后面的问题其实自然而然地就不成立了，因此对于递归处理的变量，处理时一定要合理安排其操作顺序。

5.螺旋矩阵（循环不变量）

·leetcode59：
给你一个正整数 n ，生成一个包含 1 到 n2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix 。

·坚持循环不变量：这道题不涉及什么算法，但是要特别注意的是边界的处理：左闭右开或者左闭右闭。对于这道题目而言，左闭右开是一个不错的选择。对于每一次循环的处理，处理包含最左的元素但是不包含最右的元素。
观察数字填充规律，每圈数字的填写是相同规律的，都是从左上开始，先向右，再向下，再向坐，再向上。既然每圈数字的填写是遵循规律的，因此可以将每圈的填写流程作为循环体，用总的圈数加以限制即可。n为3时要填2圈，n为4时要填2圈，n为5要填3…（每填一圈，就少两层）因此循环的次数为n/2，但是当n为奇数的时候，由于3/2＝1（向下取整），所以要手动+1（写在最后，用一个if判断即可）。解决了循环体的内容和循环次数，接下来要考虑细节，每填一圈的操作要如何使用循环进行实现呢？注意循环不变量的思想！

·为什么想到用offset？（还曾考虑过为什么offset每次加1，而不是每次加2）：
因为每次填写一行或一列的时候，我们要设置的是前进方向上的终点，而每次的终点位置都是当前行/列未被填写的前一个位置，而每填好一圈，前进方向上就会少一个格子，因此想到了offset作为已经填好圈数的计时器，并且每次加一。

6.环形链表Ⅱ

leetcode142：给定一个链表的头节点 head ，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。不允许修改 链表。

双指针：快慢指针同时从起点出发，快指针一次走两个结点，慢指针一次走一个结点。

·如何判断环？
若快慢指针能够相遇则有环，不能相遇则无环。（有环就一定会相遇吗？不会正好相错不相遇吗？不会。当慢指针进入环后，快指针开始追赶慢指针，此时快指针相对慢指针的速度是1，因此一定会相遇的）

·如何找出环的入口？

设头结点到环形入口结点为x，入口结点到快慢指针相遇的结点为y，相遇结点到环形入口结点为z，因此我们的目标就是计算出x。

在相遇时slow走过的距离为 x+y，（为什么慢指针一定会在第一圈之内被fast追上？因为当slow进环时，fast一定在环中，倘若slow走了一圈，那fast一定走了两圈，那这样的话fast一定会套住slow，因此slow在第一圈内就会被fast追上）fast走过的距离为 x+y+n(x+y)，设fast在环内走了n圈。列出时间等式：x+y=[x+y+n(x+y)]/2，整理得x=n(y+z)-y。再次整理得：x=(n-1)(y+z)+z。观察此式，当n=1时，得到x=z。当n=2时，得到x=z+（y+z），相较于n=1只是多走了一圈…

也就是说，从头结点出发的一个指针，从相遇结点也出发一个指针，这两个指针每次都只走一个结点，那么他们两个将会在环形入口结点相遇。

三、哈希表

0.哈希表理论基础

·哈希表——Hash table

哈希表是根据关键码（Key）的值而直接进行访问的数据结构

直白讲一个数组就是一张哈希表。哈希表中的关键码就是数组的索引下标，通过下标直接访问数组中的元素。

哈希表解决的问题：快速判断一个元素是否在集合中。

·哈希函数

将一组数据映射到哈希表过程中这个映射的方式就成为哈希函数。

·哈希碰撞

当通过哈希函数计算后的关键码重复，或者数据数量大于哈希表大小，就会发生哈希碰撞。

哈希碰撞有两种解决方法，拉链法和线性探测法。

拉链法

冲突的元素存储在链表中，这样就可以通过索引找到想要的元素。

拉链法要适当选择哈希表大小，这样既不会因为数组空值多而浪费内存，也不会因为链表太长导致在索引上浪费太多时间。

线性探测法

使用线性探测法一定要保证哈希表大小要大于数据量大小。当发生碰撞时，会采取特定的方法将冲突数据放在其他的空位上。

·常见的三种哈希结构

数组、set（集合）、map（映射）

数组很熟悉不再赘述。我们来看一下set。

set（集合） 底层实现 是否有序 数值是否可重复 数值能否更改 查询效率 增删效率
std::set 红黑树 有序 否 否 O(log n) O(log n)
std::multiset 红黑树 有序 是 否 O(log n) O(log n)
std::unordered_set 哈希表 无序 否 否 O(1) O(1)

红黑树是一种平衡二叉搜索树，所以key值是有序的，且key值不可以修改，改动key值会导致整棵树的错乱，因此之能增删。

接下来是map。

map（映射） 底层实现 是否有序 数值是否可重复 数值能否更改 查询效率 增删效率
std::map 红黑树 key有序 key不可重复 key不可修改 O(log n) O(log n)
std::multimap 红黑树 key有序 key可重复 key不可修改 O(log n) O(log n)
std::unordered_map 哈希表 key无序 key不可重复 key不可修改 O(1) O(1)

由于std::map和std::multimap的底层实现都是红黑树，因此其key值也是有序的（这个问题也经常作为面试题，考察对语言容器底层的理解）。

当使用集合来解决哈希问题时，优先使用unordered_set，因为其查询和增删效率最好，如果集合有序，就用set，如果要求不仅有序，还要有重复数据的话，那么就用multiset。

map中，map是一个key value的数据结构，其中对key是有限制的，对value没限制，因为key的存储方式使用的是红黑树。

其他语言例如：java里的HashMap ，TreeMap 都是一样的原理。可以灵活贯通。其中HashMap要保证key不能重复，TreeMap中key值有序且不能重复。（HashSet、TreeSet同理）

尽管例如map和set是红黑树实现，但是给我们的使用方式仍然是哈希法的使用方式，即key和value。所以使用这些数据结构来解决映射问题的方法，我们依然称之为哈希法。

·总结

当我们遇到了要快速判断一个元素是否出现集合里的时候，就要考虑哈希法。

哈希法是牺牲空间换取时间，我们使用额外的空间来存放数据才能实现快速查询。

1.有效的字母异位词

·leetcode242：给定两个字符串 *s* 和 *t* ，编写一个函数来判断 *t* 是否是 *s* 的字母异位词。注意：若 *s* 和 *t* 中每个字符出现的次数都相同，则称 *s* 和 *t* 互为字母异位词。

·思路：判断一个元素是否出现在集合中-哈希表法解题。由于a-z小写字母的ASCII编码是连续的，且个数较少，因此可以使用数组作为哈希表 。0-25分别代表a-z，遍历第一串字符获得每个字母的个数情况，再遍历第二串字符，使之相减（在一个数组上操作即可），若为全零即为字母异位词。

使用哈希法先考虑数组，如果哈希值比较大再考虑set和map

2.两个数组的交集

·思路：在最初此题的描述nums数组有可能很大，后来leetcode声明nums小于1000。在更改后可以使用数组来作为哈希表，但是在这里我们仍然认为nums数组有可能很大，而且如果哈希值比较少、特别分散、跨度非常大，使用数组就造成空间的极大浪费。不适合使用数组作为哈希表。因此使用set来解决这道问题。我们遍历nums1，将其转化为哈希表，再遍历nums2，查看元素是否在哈希表中，如果存在，就将其放入result集合中。
在集合这种哈希结构中还分为set、unordered_set、multiset，其中set和multiset底层为红黑树，unordered_set为哈希表，因此unordered_set在做映射、取值的时候是最快的，而其他两种底层是树，还有一个查找的过程。

3.快乐数

·leetcode202：编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」 定义为：

  • 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
  • 然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
  • 如果这个过程 结果为 1，那么这个数就是快乐数。

如果 n 是 快乐数 就返回 true ；不是，则返回 false 。
$$
输入：n = 19
\输出：true
\解释：
1^2 + 9^2 = 82
\8^2 + 2^2 = 68
\6^2 + 8^2 = 100
\1^2 + 0^2 + 0^2 = 1
$$
·思路：首先一个解题关键是如何计算每个位置上数字的平方和。其次题目中的重要字眼：有可能无限循环始终变不到1，我们按照普遍的迭代思想，如果变成1就return true，而如何判断false呢？这就需要我们思考无限循环这个中止条件了，怎么才能让计算机知道程序正陷入无限循环呢？通过重复！当计算出来的平方和已经出现过就说明进入循环了！而判断是否重复自然而然地想到哈希表！

4.两数之和

·leetcode1（梦开始的地方）：给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。

·思路：暴力解法使用两层for循环，对于每个元素遍历后续元素（对于之前的元素，在遍历到前面的元素的时候已经计算过了）是否存在target减去该元素，这种解法慢就慢在了多了很多重复的计算，对于已经遍历的元素没有记忆（意思是清楚直接的知道有哪些元素遍历过），因此考虑使用哈希表法解决，而题目要求返回数组下标，因此需要记录两个值，一个是元素，一个是下标，所以使用Map来做哈希。由于元素是用来索引的关键，因此元素为key，下标为value。遍历nums，寻找target减去当前元素的值是否在map中出现，如果出现则说明找到了这样的数。

5.四数相加Ⅱ

·leetcode454：给你四个整数数组 nums1、nums2、nums3 和 nums4 ，数组长度都是 n ，请你计算有多少个元组 (i, j, k, l) 能满足：

  • 0 <= i, j, k, l < n
  • nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0

·思路：四个数字相加为0，我们可以利用两数之和的思想，把四个数字相加转化为两数相加。即遍历第一第二个数组，获取到所有的两数相加（a+b）的结果加入到Map中，结果即为Key，而Value即为该结果出现的次数，因为要返回的是能够满足条件的所有的元组的个数，因此需要记录这个两数相加的结果，进而在遍历第三第四个数组时每遇到一个相加为-a-b（即四个数和为0）时，ans增加的值为该两数和的出现次数，即Value。

6.三数之和

·leetcode15：给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。注意：答案中不可以包含重复的三元组。

·思路：

哈希解法：根据四数相加，很容易想到用两个for循环来得到a和b，再遍历数组，判断set中是否存在-(a+b)。这种想法是正确的，但是难在如何去重。因此这道题不适合使用哈希法

双指针解法：使用双指针法时首先要将数组排序（升序）。使用i遍历数组，分别设置一个left指针和right指针分别指向i右侧部分数组的最左端和最右端。因为数组已经排序，当我们得到了nums[i]+nums[left]+nums[right]的大小，与0比较，若比零大，说明应该再小一些才能有可能等于零，那么只能将right再向左移动才可以；若比零大，说明应该再大一些才能有可能等于零，那么只能将left向右移动才可以。解决了最基本大致解法，还有一个关键问题–去重。去重这部分的思路直接放在代码里更加直观。

7.四数之和

·leetcode18：给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述全部条件且不重复的四元组 [nums[a], nums[b], nums[c], nums[d]] （若两个四元组元素一一对应，则认为两个四元组重复）：

  • 0 <= a, b, c, d < n
  • a、b、c 和 d 互不相同
  • nums[a] + nums[b] + nums[c] + nums[d] == target

你可以按 任意顺序 返回答案 。

·思路：有了三数之和的基础，四数之和可以在三数之和的基础上进行拓展，我们只需在三数之和的基础上再最外层再加一层循环，循环i和j，再分别遍历left和right下标。
