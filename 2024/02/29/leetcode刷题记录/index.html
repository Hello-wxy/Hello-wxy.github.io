​ 思路：该题的难点在于，当数组中不存在target，应该把target插在哪个位置上呢？（用哪个变量指示失配位置呢？）
​ 经过实践，在使用while(left<=right)时，当最终跳出循环时（失配时）right=left-1，而mid是等于left还是right取决于最后失配时nums[mid]是大于target还是小于target：当nums[mid]>target时，要更改right，而在此之前left=right=mid（失配的前一步是肯定要走到这里的，也就是while(left<=right)里=条件的最后一次）所以right=mid-1，那么此时left=mid；当nums[mid]<target时，要更改left，所以left=mid+1，right=mid；使用几组案例就可以发现，left的位置刚好就是target应该插入的位置。

2.移除元素（双指针）

·leetcode27：
·要求：删除数组中等于这个目标值的元素并且返回新数组的大小
·vector提供了erase()函数来实现删除某个位置元素的操作，其操作复杂度是O(n)的，要将后续的元素整体向前移/覆盖。与此同时，再次对该向量使用size()返回的长度会减一，但实际上，数组的实际’容量‘并没有改变，只是’长度‘被- -，最后面因为删除元素所造成的’额外‘位置只是被忽略了，没有做实际的处理。（理解 ‘容量capacity’ 和 ‘长度length/size’ 之间的差别）

①暴力解法：O(n^2)

②双指针: O(n)
快指针：新数组所需要的值
慢指针：新数组的下标
快指针不断寻找符合的元素，慢指针不断为新来的元素指明位置。有点像在编一个麻花绳，快指针在前面乱绳处不断缕齐，慢指针在后面将快指针缕齐的地方编好。

·把slow++写在nums[slow++]=nums[fast]里面会快3ms！直接击败100%用户。

4.长度最小数组（双指针、滑动窗口）

·leetcode209：
·给定一个含有 n 个正整数的数组和一个正整数 target 。找出该数组中满足其总和大于等于 target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

·暴力解法：O(n^2)

·在break这里想了很久，为什么要直接break出去呢：因为对于当前的外循环，起点i是固定的，j每次++，那么所要找的最短的一定是最先符合的子序列，所以后续j++只能使得字串变得更长了，所以应该直接break掉（有点和滑动窗口弄混了…）

·滑动窗口
在暴力解法中，第一个for循环（外层循环）滑动窗口的起始位置，第二个循环（内层循环）滑动窗口的终止位置。
而滑动窗口只使用一个for循环来完成。如果滑动窗口使用一个for循环来表示起始位置，那么只可能是一个一个不断地遍历之后的位置来获得最短子序列，那么就陷入了暴力解法的怪圈，因此一定要使用一个for循环表示滑动窗口的终止位置。
那么如何移动窗口起始位置呢？（滑动窗口的精髓）当当窗口的值大于s，那么窗口就应该缩小了（起始位置向后移动）。滑动窗口的精妙之处在于根据当前子序列的大小的情况，不断调节子序列的起始位置。

·对于滑动窗口的算法复杂度：O(n)，为啥不是o(n^2)呢?明明有两个嵌套的循环啊。
要看每个元素的操作次数，每个元素进入滑动窗口操作一次，出滑动窗口操作一次，因此每个元素操作了两次，是2 X n，因此就是O(n)。而对暴力解法而言，对于每一个起始位置，都要整个遍历终止位置，是n X n的。

·在while(sum>=s)这里想了很久，为什么有个=呢？如果等于的话则说明当sum等于s的话也要进行一次’起始位置向后移动‘的操作，那不就是说此时所得到的结果会比预期小了吗？！
例如：s=7，nums =[2,3,1,2,4,3]。最短子序列是[4,3]，最后一次4，3满足sum=s，则start++不是指向3返回1了？？
是因为我没有把while内的工作内容掌握全面，流程掌握清楚。注意看代码的实现流程，是先记录的sublength的长度，后移动的start，也就是说，当我们发现了某处满足了sum>s的条件，我们要向后移动起始位置的start了，但是移动后很有可能不满足了sum>s的条件，因此一定要首先保存subLlength然后再移动start，这样做是合理正确的，那么当sum=s时，尽管start在最后确实向后移动了一位，但是subLength的计算在start移动之前，因此并不影响最终结果的正确性。只要将前面的顺序搞明白，后面的问题其实自然而然地就不成立了，因此对于递归处理的变量，处理时一定要合理安排其操作顺序。

5.螺旋矩阵（循环不变量）

·leetcode59：
给你一个正整数 n ，生成一个包含 1 到 n2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix 。

·坚持循环不变量：这道题不涉及什么算法，但是要特别注意的是边界的处理：左闭右开或者左闭右闭。对于这道题目而言，左闭右开是一个不错的选择。对于每一次循环的处理，处理包含最左的元素但是不包含最右的元素。
观察数字填充规律，每圈数字的填写是相同规律的，都是从左上开始，先向右，再向下，再向坐，再向上。既然每圈数字的填写是遵循规律的，因此可以将每圈的填写流程作为循环体，用总的圈数加以限制即可。n为3时要填2圈，n为4时要填2圈，n为5要填3…（每填一圈，就少两层）因此循环的次数为n/2，但是当n为奇数的时候，由于3/2＝1（向下取整），所以要手动+1（写在最后，用一个if判断即可）。解决了循环体的内容和循环次数，接下来要考虑细节，每填一圈的操作要如何使用循环进行实现呢？注意循环不变量的思想！

·为什么想到用offset？（还曾考虑过为什么offset每次加1，而不是每次加2）：
因为每次填写一行或一列的时候，我们要设置的是前进方向上的终点，而每次的终点位置都是当前行/列未被填写的前一个位置，而每填好一圈，前进方向上就会少一个格子，因此想到了offset作为已经填好圈数的计时器，并且每次加一。

6.环形链表Ⅱ

leetcode142：给定一个链表的头节点 head ，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。不允许修改 链表。

双指针：快慢指针同时从起点出发，快指针一次走两个结点，慢指针一次走一个结点。

·如何判断环？
若快慢指针能够相遇则有环，不能相遇则无环。（有环就一定会相遇吗？不会正好相错不相遇吗？不会。当慢指针进入环后，快指针开始追赶慢指针，此时快指针相对慢指针的速度是1，因此一定会相遇的）

·如何找出环的入口？

设头结点到环形入口结点为x，入口结点到快慢指针相遇的结点为y，相遇结点到环形入口结点为z，因此我们的目标就是计算出x。

在相遇时slow走过的距离为 x+y，（为什么慢指针一定会在第一圈之内被fast追上？因为当slow进环时，fast一定在环中，倘若slow走了一圈，那fast一定走了两圈，那这样的话fast一定会套住slow，因此slow在第一圈内就会被fast追上）fast走过的距离为 x+y+n(x+y)，设fast在环内走了n圈。列出时间等式：x+y=[x+y+n(x+y)]/2，整理得x=n(y+z)-y。再次整理得：x=(n-1)(y+z)+z。观察此式，当n=1时，得到x=z。当n=2时，得到x=z+（y+z），相较于n=1只是多走了一圈…

也就是说，从头结点出发的一个指针，从相遇结点也出发一个指针，这两个指针每次都只走一个结点，那么他们两个将会在环形入口结点相遇。

三、哈希表

0.哈希表理论基础

·哈希表——Hash table

哈希表是根据关键码（Key）的值而直接进行访问的数据结构

直白讲一个数组就是一张哈希表。哈希表中的关键码就是数组的索引下标，通过下标直接访问数组中的元素。

哈希表解决的问题：快速判断一个元素是否在集合中。

·哈希函数

将一组数据映射到哈希表过程中这个映射的方式就成为哈希函数。

·哈希碰撞

当通过哈希函数计算后的关键码重复，或者数据数量大于哈希表大小，就会发生哈希碰撞。

哈希碰撞有两种解决方法，拉链法和线性探测法。

拉链法

冲突的元素存储在链表中，这样就可以通过索引找到想要的元素。

拉链法要适当选择哈希表大小，这样既不会因为数组空值多而浪费内存，也不会因为链表太长导致在索引上浪费太多时间。

线性探测法

使用线性探测法一定要保证哈希表大小要大于数据量大小。当发生碰撞时，会采取特定的方法将冲突数据放在其他的空位上。

·常见的三种哈希结构

数组、set（集合）、map（映射）

数组很熟悉不再赘述。我们来看一下set。

set（集合） 底层实现 是否有序 数值是否可重复 数值能否更改 查询效率 增删效率
std::set 红黑树 有序 否 否 O(log n) O(log n)
std::multiset 红黑树 有序 是 否 O(log n) O(log n)
std::unordered_set 哈希表 无序 否 否 O(1) O(1)

红黑树是一种平衡二叉搜索树，所以key值是有序的，且key值不可以修改，改动key值会导致整棵树的错乱，因此只能增删。

接下来是map。

map（映射） 底层实现 是否有序 数值是否可重复 数值能否更改 查询效率 增删效率
std::map 红黑树 key有序 key不可重复 key不可修改 O(log n) O(log n)

由于std::map和std::multimap的底层实现都是红黑树，因此其key值也是有序的（这个问题也经常作为面试题，考察对语言容器底层的理解）。

当使用集合来解决哈希问题时，优先使用unordered_set，因为其查询和增删效率最好，如果集合有序，就用set，如果要求不仅有序，还要有重复数据的话，那么就用multiset。

map中，map是一个key value的数据结构，其中对key是有限制的，对value没限制，因为key的存储方式使用的是红黑树。

其他语言例如：java里的HashMap ，TreeMap 都是一样的原理。可以灵活贯通。其中HashMap要保证key不能重复，TreeMap中key值有序且不能重复。（HashSet、TreeSet同理）

尽管例如map和set是红黑树实现，但是给我们的使用方式仍然是哈希法的使用方式，即key和value。所以使用这些数据结构来解决映射问题的方法，我们依然称之为哈希法。

当我们遇到了要快速判断一个元素是否出现集合里的时候，就要考虑哈希法。

哈希法是牺牲空间换取时间，我们使用额外的空间来存放数据才能实现快速查询。

4.KMP算法

·用于解决字符串匹配问题：例：

文本串：aabaabaaf
模式串：aabaaf

·暴力匹配：两层for循环，外循环遍历文本串，内循环遍历模式串，每次失配就让模式串右移一位继续匹配，时间复杂度为O(m*n)，空间复杂度为O(1)。
暴力匹配为什么慢呢？因为暴力匹配中多了很多回溯，每一次失配都要从模式串的最开始重新进行匹配。

·KMP：当在文本串的第二个b、模式串的f处失配时，模式串会直接跳到b，与文本串的第二个b处进行匹配。
KMP如何找到的这个b呢？KMP的精妙之处在于该算法能够“知道”我们当前已经匹配了哪些字符，让我们之前已经匹配所做的工作不白费。KMP算法使用空间换时间。

最长相等前后缀：
前缀：包含首字母不包含尾字母的所有子串；后缀：包含尾字母不包含首字母的所有子串
举一个例子：字符串abcdab
前缀集合：{a,ab,abc,abcd,abcda} 后缀集合：{b,ab,dab,cdab,bcdab} 显然，最长的相等前后缀就是ab
另一个例子：字符串ab ab ab ab
答案是：ab ab ab
前缀表：字符串每个位置处的最长相等前后缀长度：aabaaf的前缀表为[0,1,0,1,2,0]

如何使用前缀表进行匹配呢？
仍使用最开始的例子，当匹配到文本串第二个b，模式串的f时失配，这时候寻找f前面的子串（aabaa）的最长相等前后缀（前缀表：[0,1,0,1,2,0]），也就是2，因为我们希望能够直接匹配到与其相等的前缀的后面一位进行匹配，由于这个前缀表中最长相等前后缀长度刚好为相等前缀的后一位（下标索引从零开始），因此我们就能够直接索引到b处继续匹配。

next数组：
next数组用来存放字串的最长相等前后缀的长度，next数组的数值仅与字符串本身有关。next[i]=j的含义为：下标i位置处前的字串的最长相等前后缀长度为j（注不同的KMP实现对next数组的定义稍有不同，但仅是代码设计上的差别，原理相同，该定义对应着将前缀表的next整体向右移一位，首位为-1：[-1,0,1,0,1,2]；如果不这样操作还可以不右移，但是比较的时候要使用前一位的next值，相当于next[i-1]=j）

复杂度分析：
n为文本串长度，m为模式串长度，在匹配过程中只便利了一遍文本串，所以时间复杂度是O(n)，求解next数组时遍历了模式串，时间复杂度为O(m)，因此总的时间复杂度为O(m+n)。

直接给个结论：一个有重复子字符串组成的字符串减去最长相等前后缀后，剩余部分即为最小的重复单元。

步骤四：循环往复。

正是因为 最长相等前后缀的规则，当一个字符串由重复子串组成的，最长相等前后缀不包含的子串就是最小重复子串。

2.二叉树的迭代遍历

使用迭代的方法解决二叉树的前中后序递归：

·思路：递归的实现是：每一次递归调用都会把函数的局部变量，参数值和返回地址等压入栈中，然后递归返回的时候，从栈顶弹出上一次递归的各项参数，所以这就是递归为什么可以返回上一层位置的原因。因此我们是可以使用栈来实现二叉树的迭代遍历的。

前序：前序：中-左-右，由于栈是先进后出的，因此要想左在右前被遍历，就需要将右先入栈，左后入栈，并且空节点是不入栈的，即中-右-左。整体的操作逻辑为：将根节点入栈，首先将栈顶弹出，再将右节点入栈，再将左节点入栈，持续直到栈空。

中序：中序：左-中-右，但是中序遍历具有特殊性，其所需访问节点和处理节点并不相同，因此不能在前序遍历的基础上进行更改就能实现。其处理逻辑为：首先使用node用于记录当前所要处理的节点，若该节点非空则入栈，并向左迭代，若节点为空则弹出，将该节点放入结果集中，再向右迭代。

中序的整个处理逻辑其实就是完全契合着左-中-右的处理顺序，先一直向左，直到空，然后再弹出中（其实最左的叶子节点也算作一个中，因为其左孩子是空），然后继续迭代右。

后序：后序：左-右-中，由于前序为 中-左-右，我们把左右调换变为中-右-左，再将结果反转，即为左-右-中。

整体做下来，感觉中序遍历的迭代实现仍然遵循着左-中-右的处理顺序，而前序和后序遍历的迭代实现都要因为栈的先进后出特性而改变处理顺序。

0.回溯算法理论基础

·回溯法是一种搜索方式，又称为回溯搜索法。回溯是递归的副产物，只要有递归就会有回溯。回溯法的效率并不高，回溯的本质是穷举。

·回溯法解决的问题：

  • 组合问题：N个数里按照规则找出k个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N皇后，解数独等

·如何理解回溯：

回溯法解决的问题可以抽象为树：回溯解决的是在集合中递归查找子集，集合的大小够成树的宽度，递归的深度就构成了树的深度。

·回溯法解题模板：

与递归三部曲类似，for循环即为横向遍历，递归就是纵向递归。

0.贪心算法理论基础

·什么是贪心？
贪心的本质是选择每一阶段的局部最优，从而达到全局最优。

·什么时候使用贪心算法？
贪心算法没有一般性的解题模板，因此如何从局部最优推到到全局最优就是一个难点。

·贪心的一般解题步骤？

  • 将问题分解为若干个子问题
  • 找出适合的贪心策略
  • 求解每一个子问题的最优解
  • 将局部最优解堆叠成全局最优解

·leetcode376：如果连续数字之间的差严格地在正数和负数之间交替，则数字序列称为 摆动序列 。第一个差（如果存在的话）可能是正数或负数。仅有一个元素或者含两个不等元素的序列也视作摆动序列。

  • 例如， [1, 7, 4, 9, 2, 5] 是一个 摆动序列 ，因为差值 (6, -3, 5, -7, 3) 是正负交替出现的。
  • 相反，[1, 4, 7, 2, 5] 和 [1, 7, 4, 5, 5] 不是摆动序列，第一个序列是因为它的前两个差值都是正数，第二个序列是因为它的最后一个差值为零。

子序列 可以通过从原始序列中删除一些（也可以不删除）元素来获得，剩下的元素保持其原始顺序。

给你一个整数数组 nums ，返回 nums 中作为 摆动序列 的 最长子序列的长度 。

·思路：考虑所有的情况（建议画一个图出来，包含所有的情况）：1-17-5-10-10-15-15-10-5-16-8

我们使用preDiff和curDiff来记录当前点的前差值和后差值，对于17这个点，preDiff=16，curDiff=-12.curDiff=nums[i+1]-nums[i]，preDiff根据curDiff来变换

需要进行长度++的情况：
①顶峰和谷底：17为顶峰，5为谷底（相对），因此这两个位置都需要进行长度++，而这二者满足的条件为：preDiff>0&&curDiff<0 || preDiff<0&&curDiff>0
②顶峰和谷底为平坡，这种情况下我们只要保留一个位置进行长度的++，也就是第一个位置或者最后一个位置：最后一个位置的条件为preDiff=0&&curDiff>0或preDiff=0&&curDiff<0，因此①②合在一起就是preDiff>=0&&curDiff<0 || preDiff<=0&&curDiff>0

不需要进行长度++的情况：
③单调坡上：单调坡上无需进行长度++，而单调坡上preDiff和curDiff符号相同
④单调坡上出现平坡：由于②的条件，会使得单调坡上的平坡最后一处也进行++。因为在此之前我们的preDiff的更新操作是实时跟进curDiff的，因此一旦遇到了curDiff=0，下一处的preDiff就为0，倘若这时出现curDiff不为0就会++，因此我们改变preDiff的更新时机：只有在进行++操作时才更新preDiff为curDiff，因为++操作时curDiff永不为0，因此破坏掉了单调平坡时误判进行++操作时的前置条件。这样做同时也不会影响其他操作的判断，因为单调变化时，preDiff的正负是固定的，无需变化。

1.斐波那契数

·leetcode509：斐波那契数 （通常用 F(n) 表示）形成的序列称为 斐波那契数列 。该数列由 0 和 1 开始，后面的每一项数字都是前面两项数字的和。也就是：

给定 n ，请计算 F(n) 。

·思路：可以使用动态规划解决，在题目中就给出了递推公式。
①确定dp数组以及下标的含义：dp[i]代表第i个斐波那契数的值
②确定递推公式：dp[i]=dp[i-1]+dp[i-2];
③dp初始化：dp[0]=0,dp[1]=1
④确定遍历顺序：从前向后遍历
⑤打印dp（debug用）

然而本题由于dp[i]只与前两个值有关，因此无需维护数组，只用三个变量即可解决。（状态压缩）

2.爬楼梯

·leetcode70：假设你正在爬楼梯。需要 n 阶你才能到达楼顶。每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

·思路：一次可以跳1或2个台阶，因此第i步阶梯可以从第i-1跳一步，第i-2跳两步上来，因此第i步阶梯的方法是i-1、i-2阶梯方法数量的总和。由于第i步只与前两步阶梯相关，因此可以使用3个变量解决。
①确定dp数组以及下标的含义：dp[i]代表到达第i个台阶的所有方法数量
②确定递推公式：dp[i]=dp[i-1]+dp[i-2];
③dp初始化：dp[1]=1，dp[2]=2
④确定遍历顺序：从前向后遍历
⑤打印dp（debug用）

3.使用最小花费爬楼梯

·leetcode746：给你一个整数数组 cost ，其中 cost[i] 是从楼梯第 i 个台阶向上爬需要支付的费用。一旦你支付此费用，即可选择向上爬一个或者两个台阶。你可以选择从下标为 0 或下标为 1 的台阶开始爬楼梯。请你计算并返回达到楼梯顶部的最低花费。

·思路：基础的爬楼梯加上了花费：注意理解题目，题目中的例子说明了从15跳2格到顶部，说明我们要跳到cost数组的最大下标的下一个截止。第i阶的最小花费是第i-1阶加上从第i-1阶起跳的花费和第i-2阶加上从第i-2阶起跳的花费之间的较小者。
①确定dp数组以及下标的含义：dp[i]代表到达第i个台阶的最小花费
②确定递推公式：dp[i]=Math.min(dp[i-1]+cost[i-1],dp[i-2]+cost[i-2]);
③dp初始化：题目中说明可以从第0或第1阶起跳，起跳并不花费体力，因此dp[0]和dp[1]都为0
④确定遍历顺序：从前向后遍历
⑤打印dp（debug用）

4.不同路径

·leetcode62：一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。问总共有多少条不同的路径？

·思路：由于机器人只能向下或向右移动，因此当前位置的状态是由该位置的上方和左方决定的，因此dp [ i ] [ j ]代表在(i , j)处的总路径，因此递推公式为dp[ i ] [ j ]=dp[ i-1 ] [ j ]+ dp[ i ] [ j-1 ]

5.不同路径 II

·leetcode63：给定一个 m x n 的整数数组 grid。一个机器人初始位于 左上角（即 grid[0][0]）。机器人尝试移动到 右下角（即 grid[m - 1][n - 1]）。机器人每次只能向下或者向右移动一步。网格中的障碍物和空位置分别用 1 和 0 来表示。机器人的移动路径中不能包含 任何 有障碍物的方格。返回机器人能够到达右下角的不同路径数量。测试用例保证答案小于等于 2 * 109。

·思路：和不同路径Ⅰ相似，dp [ i ] [ j ]代表在(i , j)处的总路径，因此递推公式为dp[ i ] [ j ]=dp[ i-1 ] [ j ]+ dp[ i ] [ j-1 ]，但如果此处为障碍的话则dp [ i ] [ j ]为0；初始化时需要注意，障碍后面都是无法到达的，因此为0，第一次做的时候只将障碍物处设置成了0；

6.整数拆分

·leetcode343：给定一个正整数 n ，将其拆分为 k 个 正整数 的和（ k >= 2 ），并使这些整数的乘积最大化。返回 你可以获得的最大乘积 。

·思路：拆成两个数，内外两层循环i和j，两个数的乘积为j * (i-j)，dp[i]代表i的最大拆分乘积，因此dp[i]至少已经被拆分成两个数。
