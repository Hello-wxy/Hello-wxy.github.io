个人博客搭建教程参考

发表于 2024-02-29 更新于 2024-03-26 分类于 教程 阅读次数：
有关如何搭建个人博客的相关教程指南，感谢各位大佬的教程。

0.如何搭建个人博客？

​ 搭建个人博客从应用层的角度来说由大致四个部分组成：1.博客生成框架 2.文件托管平台 3.站点部署服务 4.访问加速服务（其中只有前三项为必要内容，最后一项为锦上添花）

​ 1.博客生成框架：博客生成框架是一个应用程序，负责将用户的输入转化为一个网站目录，由一些html文件、css文件以及js文件组成的目录结构（前端三件套）。Hyper Text Markup Language，超文本标记语言，从语意的角度搭建网页结构，是前端的结构层；Cascading Style Sheet，层叠样式表，从美观的角度修饰页面样式，是前端的样式层；Java Script，一种脚本语言，从交互的角度去描述页面行为，是前端的行为层。一个博客框架所要达到的效果就是，作为用户的我们能够以一种相对简单的方式进行博客的设置和书写，框架负责把这些简单的东西转化成一个完整的复杂的前端页面。在相关教程中使用的框架为Hexo，通过Hexo，我们可以用yml（.yml配置文件）来进行博客的设置，通过Markdown进行博客的书写，hexo会帮助我们把这些内容转化成一个完整的网站目录，存放在public文件夹中。

​ 2.文件托管平台：其作用是能够让我们吧文件存储在云端，并提供相应的访问和修改接口，相关教程使用的文件托管平台为github访问和修改的接口是git。

​ 3.站点部署服务：就是将已成型的网站目录部署到互联网上，让网络上的其他主机能够正常访问，最简单的是github自带的github pages，但是这个方式在国内的访问速度极慢，在相关教程中使用netlify作为部署站点的服务提供商，而把github仅作为一个托管文件的场所，这样就能享受到netlify本身的国外加速服务了。此外，netlify除了直接部署静态网站的功能之外，还支持动态的构建网站，也就是我们可以自定义一个构建网站的脚本，比如说我们后面会使用的npm run clean和npm run built这个脚本，它会先清除之前已经构建好的静态网站，然后重新从头构建，以防某些地方更新不到位，之后我们每次向github commit的时候netlify都会自动pull并执行上述的脚本。

​ 4.访问加速服务：原理是CDN加速，Contnent Delivery Network，内容交付网络，其原理是通过多结点的缓存来提高网络内容的解析和访问速度，利用的是页面访问的局部性：刚刚访问过的页面很可能再过不久就会再次访问，其工作过程是，客户端先访问离自己最近的DNS（Domain Name System）服务器结点以获取目标内容所在的结点信息，然后根据该信息去询问CDN服务器结点，由CDN负责从网络中寻找远端的实际存储内容的服务器进行内容的获取，然后将获取的内容缓存下来，这样下次访问就会比较迅速了，从而实现了对网络内容交付的加速，这里需要DNS服务器和CDN服务器相互配合，因此他们一般是同一个服务器提供商的。

​ 在相关教程的推荐中，提供的解决方案是Hexo + Github + Netlify ( + Cloudflare)
