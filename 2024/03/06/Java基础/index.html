JavaSE：

1.Java入门：

java介于编译型语言和解释型语言之间。编译型语言如c、c++，代码是直接编译成机器码执行，但是不同的平台（x86、ARM等）CPU指令集不同，因此，需要编译出每一种平台的对应机器码。解释型语言如Python、Ruby没有这个问题，可以有解释器直接加载源代码然后运行，代价是效率低。而java是将代码编译成一种“字节码”，它类似于抽象的CPU指令，然后，针对不同的平台编写虚拟机，不同平台的虚拟机负责加载字节码并执行，这样就实现了“一次编写，到处运行”的效果。

类型转换：

①自动类型转换：
范围小的变量可以直接赋值给范围大的变量，如byte可直接赋值给int。

②表达式自动类型转换：
在表达式中范围小的类型变量会自动转换成范围大的类型变量（表达式最终结果由最高范围类型决定）
byte、short、char -> int -> long -> float -> double
在表达式中，byte、short、char是直接转换成int类型参与运算的

③强制类型转换：
大范围类型 转换为 小范围类型 需要进行强制类型转换
ALT+Enter 可自动进行强制类型转换
int除法得到的还是int，小技巧：1.0 * int / int 即可得到double

静态方法为什么不能调用非静态成员？

静态方法是属于类的，在类加载的时候就会分配内存，可以通过类名直接访问。而非静态成员属于实例对象，只有在对象实例化之后才存在，需要通过类的实例对象去访问。

在类的非静态成员不存在的时候静态方法就已经存在了，此时调用在内存中还不存在的非静态成员，属于非法操作。

面向对象三大特征：封装、继承、多态

封装继承好理解，多态相对抽象，什么是多态？举个例子：你开了一家宠物店，你下达了一个通告，咱们店可以给所有哈士奇洗澡[ shower(哈士奇) ]，于是附近的居民都带着哈士奇来洗澡。但是又居民带着金毛[ shower(金毛) ]，萨摩耶[ shower(萨摩耶) ]，这些狗狗都不能洗，因为你下达的命令是给哈士奇洗澡，你发现自己做了一件傻事，明明是宠物店，却只能给哈士奇一种狗狗洗澡，于是你决定让各种宠物过来洗澡，于是你定义了各种各样的方法[ shower(哈士奇); shower(金毛); shower(萨摩耶) ]，包括给哈士奇洗澡的方法，给金毛洗澡的方法，给萨摩耶洗澡的方法，但是这样做实在是太麻烦了！于是你决定改变，只要是宠物过来就能洗澡[ shower(Animal a) //哈士奇，金毛，萨摩耶继承了Animal类]。

多态的条件：
1.要有继承
2.父类引用指向子类对象
