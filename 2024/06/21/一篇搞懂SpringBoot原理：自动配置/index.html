bean作用域：

在上述的操作中，我们通过三种方法获取了bean对象，调用print函数打印三种方法获得的bean对象可以发现，这三种方法所获取的bean对象为同一个对象(输出为同一地址)，因此bean对象在spring中默认为单例的。bean的单例还是多例取决于bean的作用域。

Spring支持五种作用域，后三种在web环境才生效：

作用域 说明
singleton 容器内同名称的bean只有一个实例（单例）（默认）
prototype 每次使用该bean时会创建新的单例（非单例）
request 每个请求范围内会创建新的实例（web环境中，了解）
session 每个会话范围内会创建新的实例（web环境中，了解）
application 每个应用范围内会创建新的实例（web环境中，了解）

第三方bean：

·如果要管理的bean对象来自于第三方(不是自定义的)，是无法使用@Component及衍生注解声明bean的，就需要用到@Bean注解

SpringBoot原理：

Spring相比于SpringBoot的缺点是：依赖配置繁琐、大量配置

起步依赖：

只需spring-boot-starter-web一个依赖，其余常用的起步依赖都内嵌在这个依赖中，通过maven的依赖传递得以实现。

自动配置：

·SpringBoot的自动配置就是当spring容器启动后，一些配置类、bean对象就自动存入到了IOC容器中，不需要我们手动去声明，从而简化开发，省去了繁琐的配置操作。

·@SpringBootApplication：

该注解在标识在SpringBoot工程引导类上，是SpringBoot中最最重要的注解，该注解由三部分组成：
1.@SpringBootConfiguration：该注解与@Configuration注解作用相同，用来声明当前类也是一个配置类。
2.@ComponentScan：组件扫描，默认扫描当前引导类所在包及其子包。
3.@EnableAutoConfiguration：SpringBoot实现自动化配置的核心注解。
注：在@EnableAutoConfiguration注解中封装着@Import注解，@Import注解中指定着一个ImportSelector接口的实现类，这个实现类中实现了ImportSelector的一个方法叫做selectimports()，这个方法的返回值是一个String数组，数组中封装着要导入到Spring IOC容器中的类的全类名。这个方法中加载了两个文件，一个是spring.factories，另一个是spring目录下的org.springframework.boot.autoconfigure.AutoConfiguration.imports。（spring.factories是spring早期自动配置加载的文件，在2.7.0版本后提供了一份全新的自动配置文件且二者兼容，在3.0.0版本之后spring.factories被废除）

通常在starter的依赖传递下，都有着autoconfigure的依赖，spring.factories文件就在这个jar包下

在左侧的外部库中可查看jar包中的内容。可以找到autoconfigure的jar包，找到spring.factories，可以看见里面都是一些类的全类名，另一个文件在spring目录下，里面放置的都以AutoConfiguration结尾的自动配置类的全类名。这份文件最终会被读取，并且通过import注解加载到IOC容器中。

·@Conditional：条件装配

作用：按照一定的条件进行判断，在满足给定条件后才会注册对应的bean对象到Spring IOC容器中。
位置：方法、类

自定义starter案例：

应用场景：在实际开发中，经常会定义一些公共组件，提供给各个项目团队使用。而在SpringBoot的项目中，一般会将这些公共组件封装为SpringBoot的starter中。
在这个过程中需要两个包：starter包和autoconfigure包，starter包负责依赖管理(将所要用到依赖引入进来)，autoconfigure包负责自动配置功能，在starter中要引入这个autoconfigure包。

案例需求：自定义aliyun-oss-spring-boot-starter，完成阿里云OSS操作工具类AliyunOSSUtils的自动配置。
目标：引入起步依赖后，要想使用阿里云OSS，注入AliyunOSSUtils直接使用即可。
步骤：1.创建aliyun-oss-spring-boot-starter的maven模块
2.创建aliyun-oss-spring-boot-autoconfigure模块，在starter中引入该模块
3.在aliyun-oss-spring-boot-autoconfigure模块中定义自动配置功能，并定义自动配置文件META-INF/spring/xxx.imports

AliOSSUtils中报错：
1.MultipartFile：从前文import可知其为web中的依赖，因此要在yml中加入web开发相关依赖：

2.缺失aliOSSProperties的set/get方法：

查看AliOSSProperties，发现其使用了lombok的依赖，对此我们可以直接加入set/get方法，不再引入lombok依赖。

手动添加set/get方法：alt+insert，选择Getter和Setter

添加完成后，目前来看两个类不再报错。但是细看AliOSSProperties类仍然存在问题：由于aliyun-oss-spring-boot-autoconfigure永远不会被spring组件扫描，因此@Component注解是无效的，删除。删除后@ConfigurationProperties(prefix = “aliyun.oss”)这个注解会报错，稍后处理。同样的，在AliOSSUtils中所使用的@Component和@Autowired注解同样是无效的，删除。

③定义自动配置类：因此我们需要定义自动配置类，实现自动配置。创建AliOSSAutoConfiguration配置类，为了使得引入该starter依赖的工程能够使用Utils，要将AliOSSUtils声明为bean。

由于在AliOSSUtils中要调用AliOSSProperties，并且会使用其get/set方法，但是我们在声明aliOSSProperties后并未对其进行赋值操作，因此我们需要定义针对AliOSSProperties的set/get方法。

并且我们要在AliOSSAutoConfiguration中的return之前为AliOSSProperties赋值，然而，我们要赋值的对象aliOSSProperties从何而来呢？

这个对象是Sring加载配置文件中aliyun.oss前缀的配置项最终封装成的bean对象，而目前该处仍在报错，原因就是这里已经不再是一个bean了，在之前我们将@Component注解删除掉了。我们把鼠标悬浮在报错上，就可以得知该bean对象没有通过@EnableConfigurationProperties注册，该注解只能加在声明@Bean的方法上和@Configuration类上，因此我们可以将该注解加在配置类上。

我们通过@EnableConfigurationProperties将aliOSSProperties加入到了IOC容器中，我们需要这个bean该怎么办呢？我们之前学习过，在声明第三方bean时，要注入某个对象，只要将该bean作为方法形参传递即可，其会根据类型进行自动装配。

最后一步：要想这个配置类被加载，就需要在一份固定的配置文件中来配置，springboot在启动时就会加载这个配置文件。在分析@SpringBootApplication这个注解的源码时我们得知在org.springframework.boot.autoconfigure.AutoConfiguration.imports文件中存放着自动配置类的全类名，因此我们要在resources目录下创建两级目录：META-INF/spring，在其中创建一个文件，名称即为org.springframework.boot.autoconfigure.AutoConfiguration.imports，并将AliOSSAutoConfiguration该配置类的全类名加入其中即可（右键配置类类名，在复制中找到复制引用即可复制全类名）。
